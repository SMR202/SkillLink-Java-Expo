how to do test, do it on the actual software?10:44 AMDeciphered practical testing execution guidance for Sprint deliverablesDeciphered practical testing execution guidance for Sprint deliverablesShort answer: yes, test on the actual software. The test cases in your deliverable aren't just for documentation — you need to actually run them and show evidence. Here's how:

Two levels of testing
1. Unit/Integration tests — written in Java (automated)
2. Manual black-box tests — done via Postman + browser

Backend — JUnit tests
Add this to pom.xml if not already there:
xml<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-test</artifactId>
    <scope>test</scope>
</dependency>
Example test for ReviewService (this is your white-box coverage):
java@SpringBootTest
class ReviewServiceTest {

    @Autowired ReviewService reviewService;
    @MockBean ReviewRepository reviewRepo;
    @MockBean BookingRepository bookingRepo;

    @Test
    void shouldThrow_whenBookingNotCompleted() {
        Booking booking = new Booking();
        booking.setStatus(BookingStatus.PENDING); // not COMPLETED
        when(bookingRepo.findById(1L)).thenReturn(Optional.of(booking));

        assertThrows(IllegalStateException.class, () ->
            reviewService.submitReview(1L, 1L, 4, "Good service overall"));
    }

    @Test
    void shouldThrow_whenDuplicateReview() {
        Booking booking = new Booking();
        booking.setStatus(BookingStatus.COMPLETED);
        when(bookingRepo.findById(1L)).thenReturn(Optional.of(booking));
        when(reviewRepo.existsByBookingIdAndClientId(1L, 1L)).thenReturn(true);

        assertThrows(DuplicateReviewException.class, () ->
            reviewService.submitReview(1L, 1L, 4, "Good service overall"));
    }

    @Test
    void shouldSave_whenValid() {
        Booking booking = new Booking();
        booking.setStatus(BookingStatus.COMPLETED);
        when(bookingRepo.findById(1L)).thenReturn(Optional.of(booking));
        when(reviewRepo.existsByBookingIdAndClientId(1L, 1L)).thenReturn(false);

        reviewService.submitReview(1L, 1L, 4, "Great experience with this provider");
        verify(reviewRepo, times(1)).save(any());
    }
}
Run all tests:
bashmvn test
This generates a report in target/surefire-reports/ — screenshot that for your deliverable evidence.

Backend — Postman (your black-box tests)
This is the main evidence you'll submit. For every test case in your document, hit the actual endpoint.
Setup:

Download Postman (free)
Create a collection called "SkillLink Sprint 3 Tests"
Set a base URL variable: {{base_url}} = http://localhost:8080
Get a JWT token first via POST /api/auth/login, save it as a collection variable

Example — TC-09-01 (valid review):
POST {{base_url}}/api/reviews
Authorization: Bearer {{token}}
Body (JSON):
{
  "bookingId": 1,
  "rating": 4,
  "reviewText": "The service was professional and completed on time."
}

Expected: 201 Created
Example — TC-09-05 (text too short):
POST {{base_url}}/api/reviews
Body:
{
  "bookingId": 1,
  "rating": 3,
  "reviewText": "Good"
}

Expected: 400 Bad Request, error = "TextTooShort"
Example — TC-11-03 (booking not accepted yet):
POST {{base_url}}/api/payments/checkout
Body:
{
  "bookingId": 5,     ← this booking is still PENDING
  "paymentMethod": "CARD",
  "paymentToken": "tok_test_123"
}

Expected: 403, error = "BookingNotAccepted"
For each test: screenshot the request + response. That's your test execution evidence.
