1. Introduction
SkillLink is a digital service marketplace platform connecting clients who need skilled professionals with service providers across trades and
expertise. It addresses the absence of a trustworthy, centralised portal in Pakistan where clients can discover, vet, and book providers such
as plumbers, electricians, graphic designers, tutors, and software developers.
2. Project Vision
Vision: To become the leading digital service marketplace in Pakistan, empowering clients to access quality services instantly while
enabling skilled providers to grow their businesses through a transparent, secure, and user-friendly platform.
•
Seamless discovery of providers across 20+ skill categories.
•
Secure, role-based environment for clients and providers.
•
End-to-end service delivery — from booking to payment to
review.
•
Trust through transparent profiles, reviews, and ratings.
3. Intended Use of the System — Stakeholders
SkillLink serves Clients (seek services), Service Providers (offer expertise), and a Platform Admin (moderation). A Dev Team maintains
the system.

6. Structured Specifications (US-03: Provider Profile Setup)
Spec ID
Spec-03.1
Description
Upload, validate, and store a profile avatar.
Pre-cond
Logged in as Service Provider.
Inputs
.jpg, .jpeg, .png image file.
Process
1. Check file format. 2. Check size vs 5 MB limit.
3. Compress and store in cloud bucket. 4.
Update DB record with image URL.
Outputs
Updated avatar confirmed on UI.
Exceptions
FileTooLarge (>5 MB). UnsupportedMediaType
(bad format).
Spec ID
Spec-03.2
Description
Tag profile with searchable service categories.
Pre-cond
Profile page loaded with active DB connection.
Inputs
Array of category IDs e.g. [ID_Plumbing].
Process
1. User selects 1-5 skills from dropdown. 2.
System maps IDs to user foreign key in DB.
Outputs
Profile metadata updated; appears in relevant
search results.
Exceptions
0 skills: UI validation error. >5: disable further
selection.
Spec ID
Spec-03.3
Description
Text area for providers to pitch services to
clients.
Pre-cond
User is on the profile edit screen.
Inputs
Alphanumeric text string.
Process
1. Real-time character counter. 2. Sanitise input
(XSS prevention). 3. Save text to DB record.
Outputs
Formatted text rendered on public provider
landing page.
Exceptions
Length < 50 chars: show Bio too short warning,
disable save.
Spec ID
Spec-03.4
Description
Validate and store external portfolio URLs.
Pre-cond
User is on the profile edit screen.
Inputs
URL string (GitHub, Behance, LinkedIn, etc.).
Process
1. Regex validates http/https format. 2. Store
verified link in profile table.
Outputs
Clickable portfolio icons/links on public profile.
Exceptions
Invalid URL: highlight input red, prompt for
correction.
